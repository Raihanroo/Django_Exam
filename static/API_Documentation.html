📦 Inventory System

API Logic Documentation

Django-based Product Management System

1. System Overview

The Inventory System is a Django-based web application designed for managing products through an approval workflow. Users can upload product data via Excel files, review drafts, and approve products for the main inventory.

Key Features:
  • Excel file upload for bulk product import
  • Draft/Approval workflow
  • Search and filtering capabilities
  • Paginated product listings

2. Data Model

3. API Endpoints

GET /

Upload & Drafts Page - Displays the upload form and list of draft products with pagination.

Query Parameters:

Parameter Type Required Description
page Integer No Page number for pagination (default: 1)

Form Data (POST):

Field Type Required Description
excel_file File Yes Excel (.xlsx) file containing product data
Response Codes: 200 OK 302 Redirect 500 Error
Excel File Format: The file must contain columns: product_id, name, category, price, quantity
POST /

Excel Upload Handler - Processes Excel file upload and creates/updates products with "Draft" status.

Processing Logic:

  1. Read Excel file using pandas library
  2. Iterate through each row in the spreadsheet
  3. Use update_or_create() to insert or update product
  4. Set status to "Draft" for all imported products
  5. Redirect back to drafts page with success message
Key Function: Product.objects.update_or_create(product_id, defaults={...})
This ensures idempotency - same product_id updates existing record instead of creating duplicate.
POST /approve//

Approve Product - Approves a specific draft product and changes its status to "Approved".

URL Parameters:

Parameter Type Description
pk Integer Primary key of the product to approve

Processing Logic:

  1. Fetch product by primary key using get_object_or_404()
  2. Update status field to "Approved"
  3. Save the product object
  4. Display success message
  5. Redirect to drafts page
Response Codes: 302 Redirect 404 Not Found
GET /approved/

Approved Products List - Displays all approved products with search and pagination.

Query Parameters:

Parameter Type Required Description
q String No Search query for filtering by name or category
page Integer No Page number for pagination

Search Logic:

Uses Django Q objects for case-insensitive search:
Q(name__icontains=query) | Q(category__icontains=query)

6. Pagination

Both draft and approved product lists use Django's Paginator with the following settings:

  • Items per page: 10 products
  • Ordering: By last_updated (newest first)
  • URL format: ?page=2
