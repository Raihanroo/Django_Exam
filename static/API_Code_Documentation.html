3. API #1: Upload & Drafts Page

GET / POST /

এই API দুটি কাজ করে:

  • GET: Draft products এর তালিকা দেখায়
  • POST: Excel file upload করে এবং products তৈরি করে

View Function Code:

1import pandas as pd 2from django.shortcuts import render, redirect, get_object_or_404 3from django.contrib import messages 4from django.core.paginator import Paginator 5from .models import Product 6from django.db.models import Q 7 8 9def upload_and_drafts(request): 10 # POST Request Handle - Excel Upload 11 if request.method == "POST" and request.FILES.get("excel_file"): 12 file = request.FILES["excel_file"] 13 try: 14 df = pd.read_excel(file) # Excel পড়া 15 for _, row in df.iterrows(): # প্রতিটি row তে loop 16 Product.objects.update_or_create( 17 product_id=row["product_id"], 18 defaults={ 19 "name": row["name"], 20 "category": row["category"], 21 "price": row["price"], 22 "quantity": row["quantity"], 23 "status": "Draft", # Default status = Draft 24 }, 25 ) 26 messages.success(request, "Excel uploaded successfully!") 27 except Exception as e: 28 messages.error(request, f"Error: {e}") 29 return redirect("upload_and_drafts") 30 31 # GET Request Handle - Show Drafts 32 draft_list = Product.objects.filter(status="Draft").order_by("-last_updated") 33 paginator = Paginator(draft_list, 10) # 10 items per page 34 page_obj = paginator.get_page(request.GET.get("page")) 35 return render(request, "products/drafts.html", {"page_obj": page_obj})
কিভাবে কাজ করে:
  1. User Excel file upload করে POST request এর মাধ্যমে
  2. Pandas library দিয়ে Excel parse করা হয়
  3. প্রতিটি row তে loop চালিয়ে update_or_create() call করা হয়
  4. যদি product_id আগে থাকে তাহলে update হয়, নাহলে নতুন create হয়
  5. সব product "Draft" status পায়
  6. Paginator 10টি করে দেখায়

4. API #2: Approve Product

POST /approve//

Draft product কে Approved করে:

View Function Code:

36def approve_product(request, pk): 37 # Product খুঁজে বের করা (404 না পেলে) 38 product = get_object_or_404(Product, pk=pk) 39 40 # Status পরিবর্তন করা 41 product.status = "Approved" 42 product.save() 43 44 # Success message দেখানো 45 messages.success(request, f"Product {product.product_id} approved!") 46 return redirect("upload_and_drafts") 47 # drafts page এ ফিরে যাওয়া
কিভাবে কাজ করে:
  1. URL থেকে pk (primary key) পাওয়া যায়
  2. get_object_or_404(Product, pk=pk) - product খোঁজে, না পেলে 404 error
  3. product.status = "Approved" - status পরিবর্তন
  4. product.save() - database এ save
  5. Success message সহ redirect করে
URL এ pk কেন?
pk হলো Django এর automatic primary key। URL: /approve/1/ মানে product id=1 কে approve করা।

5. API #3: Approved Products List

GET /approved/

সমস্ত Approved products দেখায় সার্চ এবং pagination সহ:

View Function Code:

48def approved_products(request): 49 # Search query পাওয়া 50 query = request.GET.get("q", "") 51 52 # সব Approved products নেওয়া 53 products = Product.objects.filter(status="Approved").order_by("-last_updated") 54 55 # যদি search query থাকে 56 if query: 57 products = products.filter( 58 Q(name__icontains=query) | Q(category__icontains=query) 59 ) 60 61 # Pagination 62 paginator = Paginator(products, 10) 63 page_obj = paginator.get_page(request.GET.get("page")) 64 return render(request, "products/approved.html", {"page_obj": page_obj})
কিভাবে কাজ করে:
  1. ?q=search query parameter পড়া হয়
  2. শুধু status="Approved" products ফিল্টার করা হয়
  3. Search থাকলে Q() object দিয়ে name বা category এ search করা হয়
  4. __icontains = case-insensitive match (যেকোনো case এ match করে)
  5. Pagination apply করে ১০ পার পেজ দেখায়

Search Query Examples:

URL Result
/approved/ সব approved products
/approved/?q=shirt "shirt" নামে products
/approved/?q=electronics Electronics category

6. Complete Request-Response Flow

Excel Upload Flow:

Browser POST / → Django View → Pandas → Database → Response

Approve Product Flow:

Browser POST /approve/1/ → Django get_object_or_404 → Update Status → Save → Redirect

View Approved Products Flow:

Browser GET /approved/?q=search → Django Filter(Q) → Paginator → Render Template → HTML Response
